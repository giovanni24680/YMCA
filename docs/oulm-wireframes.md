# Oulm Low-Fi Board Spec

This file mirrors the intended single-board Excalidraw wireframe structure so the low-fidelity step is easy to recreate or hand off.

## Board Lanes

Use one board with left-to-right flow and a thin top sitemap lane.

### Lane 1: Sitemap

- Homebase
- Events
- Event detail
- RSVP
- Onboarding
- Map
- Host
- Booking wizard
- Progress
- Partners

### Lane 2: Homebase Flow

- Hero with third-place framing
- Why showing up is hard
- Featured events strip
- County anchors strip
- Progress / partner teasers

### Lane 3: Event Activation

- Events list with filters
- Event detail page
- Dual CTA:
  - RSVP softly
  - Show up directly
- Social proof and FAQ blocks

### Lane 4: RSVP and Onboarding

- RSVP intent chips
- guest count choices
- optional arrival note
- soft account choices
- onboarding quick-pick clouds
- suggested circles output

### Lane 5: Map

- county-level map
- Homebase HQ highlighted
- flagship and community spaces
- venue modal
- event link out from modal

### Lane 6: Host and Booking

- host overview
- intentional friction rationale
- multi-step booking wizard
- review and pending state

### Lane 7: Progress and Pathways

- presence points
- host points
- verified hours
- external proof scaffold
- partner/resource cards

## Annotation Keys

Mark these explicitly on the board:

- `Soft friction`
- `Trust cue`
- `Locality cue`
- `Post-RSVP identity`
- `Exportable local state`

## Validation Checklist

- Mobile portrait first
- Mobile landscape pass
- Tablet portrait pass
- Laptop pass
- Kiosk readability pass
- Touch target review
- Copy tone review
- Contrast and hierarchy review
